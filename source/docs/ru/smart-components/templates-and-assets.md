# Шаблоны и ассеты Smart Components

Утверждённая поставка может содержать ассеты, необходимые для конкретных
контрактов Smart Components. Их состав не одинаков для всех элементов: запись
может иметь собственный JavaScript, использовать собственный CSS, зависеть от
Core или обычного компонента, а шаблонное поведение определяется её отдельным
контрактом.

## Что фиксировать в reference-странице

- путь к ассету в утверждённой поставке, если его определяет схема;
- контракт, который использует этот ассет;
- подтверждённые зависимости;
- наличие собственного CSS, если оно подтверждено для записи;
- шаблонное поведение и источник разметки, если оно подтверждено;
- способ проверки в браузере.

## Чего не следует предполагать

- Каждый элемент Smart Components имеет отдельный CSS-файл.
- Каждый элемент Smart Components получает внешний HTML-шаблон.
- Файлы из соседних каталогов можно подключить вручную без подтверждённых зависимостей.
- Одинаковые атрибуты и события работают у разных элементов одинаково.

Если компонент использует дочернюю разметку, шаблон или серверно подготовленные
данные, эти требования должны быть описаны в его собственной странице вместе с
примером и lock, на котором он проверен.

Проверяемая карта жизненного цикла описана в разделе
[Загрузка и зависимости](/ru/smart-components/lifecycle/), а контракт отдельных
страниц — в [справочнике](/ru/smart-components/reference/).
