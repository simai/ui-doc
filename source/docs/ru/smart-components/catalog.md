# Каталог Smart Components

Публикуемый каталог Smart Components должен собираться из утверждённой
интеграционной схемы и source evidence одной поставки. Нельзя объединять список
из одной ревизии с файлами или примерами из другой.

## Поля записи

| Поле | Назначение |
| --- | --- |
| Идентификатор контракта | Связывает запись с утверждённой схемой |
| Допустимый вход | Показывает подтверждённую разметку или конфигурацию сценария |
| JavaScript и собственный CSS | Фиксирует реально объявленные ассеты без предположения об их универсальности |
| Зависимости | Показывает связанные элементы Smart Components и обычные компоненты |
| Шаблонное поведение | Описывается только при подтверждённом source evidence |
| Пример | Указывает на пример, проверенный на том же lock |
| Проверка | Хранит browser smoke, доступность и revision evidence |

## Правила публикации

1. Добавляйте запись, только если интеграционная схема и source evidence
   относятся к одной закреплённой поставке.
2. Не включайте в публичный каталог элементы из незафиксированной ветки или
   локальной сборки.
3. Не переносите список атрибутов между разными элементами: он является частью
   контракта конкретной записи.
4. Если запись Smart Components зависит от обычного компонента, добавляйте двустороннюю
   ссылку между разделами.
5. При смене Core или Smart lock повторяйте сверку каталога и browser smoke.

Для понимания проверяемой карты жизненного цикла прочитайте
[Загрузку и зависимости](/ru/smart-components/lifecycle/). Формат отдельных
карточек описан в [справочнике](/ru/smart-components/reference/).
