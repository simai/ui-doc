# Примеры Smart Components

Исходные примеры помогают понять состав разметки и ожидаемый сценарий, но
публиковать их без проверки нельзя. Playground может использовать другую
локальную конфигурацию Core и Smart-ассетов, поэтому он не заменяет release
evidence.

## От источника к документационному примеру

1. Найдите контракт Smart Components в [каталоге](/ru/smart-components/catalog/).
2. Выберите пример, который соответствует именно этому контракту, а не только
   похож по названию.
3. Запустите его на утверждённом Core/Smart lock.
4. Оставьте минимальную разметку, необходимую для сценария.
5. Проверьте сеть, клавиатуру, фокус, состояние ошибки и relations.
6. Зафиксируйте source path и результат проверки рядом с примером.

## Практические правила

- Если имя группы playground отличается от идентификатора контракта, создайте явную
  mapping-запись в каталоге.
- Не объявляйте custom example публичным компонентом, пока для него нет
  подтверждённого контракта и поставляемых ассетов.
- Для диалогов, выпадающих списков и других интерактивных сценариев обязательно
  проверяйте управление фокусом и клавиатурой.
- Для загрузки данных или файлов добавьте проверки пустого состояния, ошибки и
  повторной инициализации, если они входят в контракт записи.

Завершённый пример должен ссылаться на конкретную страницу в
[справочнике](/ru/smart-components/reference/), а не оставаться изолированным
фрагментом.
