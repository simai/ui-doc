# Справочник Smart Components

В этом разделе публикуются отдельные страницы Smart Components после
подтверждения их контракта на утверждённой поставке. Каждая страница описывает
один подтверждённый элемент; она не должна смешивать сведения из разных ревизий
или рабочих веток.

## Обязательный состав страницы

- назначение и граница элемента;
- идентификатор и допустимый вход из утверждённой схемы;
- необходимые ассеты и подтверждённые зависимости;
- подтверждённые параметры, события и шаблонное поведение;
- минимальный пример с source path;
- доступность и browser smoke;
- Core/Smart release lock, на котором запись проверена.

Если доступно только имя файла, пример из playground или внутренний ассет, этого
недостаточно для публикации API-справки. Сначала подтвердите контракт, поставку
и поведение в браузере.

Вернитесь к [каталогу](/ru/smart-components/catalog/), чтобы выбрать запись,
или к [примерам](/ru/smart-components/examples/), чтобы подготовить
воспроизводимый сценарий.
