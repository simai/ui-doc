# Smart Components

Smart Components SIMAI Framework применяются, когда элементу интерфейса нужен
явный жизненный цикл: состояние, события, зависимости или работа с данными.
Утверждённая интеграционная схема определяет, как такой элемент обозначается,
какие ассеты и связи нужны для его сценария и как это подтверждается.

## Когда выбирать Smart Components

| Сценарий | Подходящий слой |
| --- | --- |
| Только оформление, сетка или адаптивный modifier | [Утилиты](/ru/utilities/) |
| Повторяющаяся разметка и локальное поведение | [Компоненты](/ru/components/introduction/) |
| Состояние, зависимости, события или взаимодействие с данными | Smart Components |

Запись Smart Components не является синонимом любой пользовательской разметки.
Её идентификатор, допустимый вход, зависимости и доступные параметры должны
быть подтверждены для конкретной поставки.

## Что важно знать заранее

- Идентификаторы и допустимая разметка определяются утверждённым release lock и
  интеграционной схемой; не переносите их из другой ревизии по аналогии.
- Схема может определять связи с обычными компонентами или другими Smart
  Components. Не подключайте файлы выборочно, игнорируя подтверждённые связи.
- Нельзя предполагать, что у каждого элемента Smart Components есть собственный CSS-файл,
  HTML-шаблон или одинаковый набор атрибутов. Это свойства конкретной записи.
- Примеры из playground полезны как исходный материал, но становятся
  пользовательской документацией только после проверки на lock проекта.
