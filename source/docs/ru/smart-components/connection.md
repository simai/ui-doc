# Подключение Smart Components

Smart Components подключаются как часть согласованной поставки Core и Smart
ассетов. Сначала зафиксируйте пару поставок, затем настройте пути и только после
этого переносите разметку из примера.

## Порядок подключения

1. Выберите утверждённый immutable lock для Core и Smart Components.
2. Сохраните структуру пакетов, определённую интеграционной схемой, без
   незафиксированного переименования внутренних путей.
3. Настройте entry point и базовые пути ровно так, как предписывает утверждённая
   схема.
4. Если Smart-пакет размещён отдельно, зафиксируйте это в том же lock и в той
   же интеграционной схеме.
5. Используйте минимальный пример, подтверждённый для того же lock, и проверьте
   его в браузере.

Конкретные значения путей, URL и идентификаторы поставки должны находиться в
release lock проекта. Не подставляйте вместо них ветку, плавающую версию или
значение из чужого окружения.

## Проверка после подключения

Проверьте, что:

- браузер загружает Core и Smart-ассеты без `404` и ошибок типов контента;
- страница использует подтверждённый сценарий элемента и его связи доступны;
- интерактивный элемент доступен с клавиатуры и корректно управляет фокусом;
- повторная инициализация не приводит к дублированию ассетов или обработчиков.

Если компонент использует данные, шаблоны или события, дополните эту проверку
сценарием из его reference-страницы. Общая установка описана в
[Старт → Установка](/ru/start/installation/); правила жизненного цикла — на
странице [Загрузка и зависимости](/ru/smart-components/lifecycle/).
