# Каталог компонентов

Публикуемый каталог компонентов должен строиться по утверждённой
интеграционной схеме и source evidence, а не по простому списку каталогов в
дистрибутиве. Это важно: часть файлов служит вспомогательным кодом, а часть
элементов использует зависимости из других слоёв.

## Статусы записей

| Статус | Что означает | Как использовать в документации |
| --- | --- | --- |
| Подтверждённый контракт компонента | Утверждённая схема определяет способ использования и необходимые ассеты | Добавлять в пользовательский каталог после проверки на lock |
| Связанный ассет | Файл используется контрактом или зависимостью другого элемента | Указывать как деталь реализации, не как самостоятельный API |
| Вспомогательный ассет | Файл находится в дистрибутиве, но не имеет самостоятельного пользовательского контракта | Не создавать отдельную страницу без явного source evidence |

## Обязательные поля каталога

У каждой опубликованной записи должны быть следующие поля:

| Поле | Зачем оно нужно |
| --- | --- |
| Идентификатор контракта | Связывает документацию с утверждённой схемой |
| Допустимый вход | Показывает, какая разметка или конфигурация подтверждена для элемента |
| Ассеты и зависимости | Предотвращает неполное ручное подключение |
| Пример | Ведёт к проверенной разметке, а не к предположению |
| Доступность | Фиксирует фокус, клавиатуру, имя и иные существенные требования |
| Source evidence | Хранит путь к правилу, примеру и release lock |

## Как поддерживать каталог

1. Начинайте с утверждённой интеграционной схемы и source evidence поставки.
2. Сверяйте запись с фактическим составом поставляемых ассетов.
3. Добавляйте связь с примером только после проверки его на том же lock.
4. Если у записи есть Smart-версия, связывайте страницы между разделами, но не
   смешивайте их контракты.
5. При обновлении Core пересобирайте таблицу и повторяйте browser smoke для
   затронутых примеров.

Структура будущих детальных страниц описана в
[справочнике компонентов](/ru/components/reference/). Для примеров и их статуса
проверки смотрите [отдельную страницу](/ru/components/examples/).
