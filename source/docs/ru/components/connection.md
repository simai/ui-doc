# Подключение компонентов

Подключение компонентов начинается не с копирования CSS или JavaScript из
отдельной папки, а с закреплённой поставки SIMAI Framework. Используемые
ассеты, точки подключения и связанные элементы должны быть описаны одной
утверждённой интеграционной схемой.

## Порядок подключения

1. Выберите утверждённый immutable lock для поставки, которую использует
   проект.
2. Разместите пакеты в структуре, определённой утверждённой интеграционной
   схемой. Не переименовывайте внутренние пути без её обновления.
3. Настройте entry point и базовые пути ровно так, как предписывает эта схема.
4. Если проект использует отдельную поставку Smart Components, зафиксируйте её
   в том же lock и в той же интеграционной схеме.
5. Возьмите минимальный пример из проверенного набора и откройте страницу в
   браузере. Убедитесь, что ассеты не дают `404`, а элемент и его локальное
   поведение работают.

Конкретные URL, идентификаторы и фрагменты разметки берите из release lock и из
проверенной reference-страницы, а не из ветки исходников или случайного
playground-сеанса.

## Что не следует делать

- Не подключайте `@main`, `@latest` или незафиксированную локальную сборку в
  production.
- Не копируйте один CSS- или JS-файл вручную в обход утверждённой структуры:
  компонент может требовать дополнительные зависимости.
- Не считайте, что каждый ассет каталога имеет одинаковый набор CSS, JS или
  параметров.
- Не переносите пример без проверки на lock проекта, даже если похожий пример
  открылся в playground.

## Минимальная проверка

После подключения проверьте в браузере:

- нет ли сетевых запросов с кодом `404` и ошибок инициализации;
- доступны ли CSS и JavaScript, предусмотренные утверждённой схемой;
- сохранились ли клавиатурное управление, фокус и доступное имя у интерактивных
  элементов;
- не появились ли дублирующиеся запросы к одним и тем же ассетам.

Для общей установки смотрите [Старт → Установка](/ru/start/installation/) и
[Старт → Загрузчик](/ru/start/loader/). Если элемент требует состояние или
внешние зависимости, продолжите с
[подключением Smart Components](/ru/smart-components/connection/).
