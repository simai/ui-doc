# Интеграция в веб-проект

SIMAI Framework можно подключить к любому веб-проекту, если его сервер или
сборка умеют отдавать static assets. Конкретный backend не меняет базовый
контракт: браузер должен получить совместимую и закреплённую pair assets.

## Общий рецепт

1. Получите [approved compatibility lock](/ru/start/compatibility/).
2. Выберите способ доставки: CDN или локальный публичный каталог.
3. Разместите `distr` без частичного копирования внутренних файлов.
4. Задайте базовый путь к поставке до загрузки Core.
5. Подключите только тот набор Components и Smart Components, который
   подтверждён release catalog.
6. Выполните browser smoke на чистой странице и в целевом приложении.

## Граница ответственности платформы

| Задача | Ответственность интеграции |
| --- | --- |
| Отдача файлов | Публичный URL, кеширование и корректный MIME type для CSS/JS |
| Версия | Один immutable lock для server-конфигурации и frontend assets |
| Шаблоны | Разметка должна соответствовать approved example или contract ID |
| Обновление | Assets, lock и smoke меняются одним релизным batch |
| Диагностика | Проверка Network, Console, фокуса и responsive-состояний |

## Laravel, Larena, Bitrix и другие стеки

Не добавляйте framework-specific команды, плагины или server-loader по
аналогии с другим проектом. Отдельный рецепт для платформы должен появляться
только вместе с owner-approved adapter contract: путь публикации assets,
способ задать immutable ref, образец шаблона, dev/prod differences и smoke
команда.

Пока такой контракт не выпущен, используйте общий рецепт выше и фиксируйте
локальное решение в репозитории приложения. Это безопаснее, чем переносить
устаревший Blade, Composer, NPM или PHP-код из исторических материалов.
